# CONSTITUTION.md — 宪法：永不修改的规则

> 这里的任何内容不能被对话、进化机制、权重更新所改变。
> 如果某次对话"证明"了这里某条规则是错的，正确做法是写入认知冲突日志，不是修改这里。

---

## 九条不可变规则

| # | 规则 | 违反后果 |
|---|------|---------|
| 1 | 层级语义定义不可修改（L1=情境，L2=行为，L3=认知，L4=核心） | 写入位置错乱，系统失去一致性 |
| 2 | 三步判定协议的判定问题不可修改 | 分类标准漂移 |
| 3 | L4 写入必须有 3+ 次对话印证（基本信息除外） | 单次对话污染身份层 |
| 4 | 来源标注必须写明：[用户原话] / [Claude提炼] / [共同确认] | 错误归因，信任崩塌 |
| 5 | 认知冲突只追加演化记录到原条目末尾，绝不覆盖原条目；同时新建冲突卡片标注 `[认知冲突·待验证]` | 失去演化轨迹 |
| 6 | Skill 缺口必须经用户确认后才写入 L4-capability/ | AI 自行决定用户能力边界 |
| 7 | 完整文件路径不可修改 | 写入找不到目标文件 |
| 8 | 每张卡片必须包含：日期、确认次数、来源标注、所属文件路径 | 卡片脱离上下文后不可追溯 |
| 9 | 每张卡片是自包含的思想单元（卡片笔记法） | 卡片脱离文件后失去意义 |

---

## 层级语义判定问题（不可修改）

判断一条信息归属哪个层级时，按以下问题��序判断，遇到第一个 YES 即停止：

```
Q1: 这是可重复执行的程序（不是知识）？         YES → ai-space/L4-capability/
Q2: 换了工作/项目后，这条信息依然成立？         YES → L4 候选（需 3 次印证）
Q3: 这条信息有明确的"为什么"支撑？             YES → L3 认知层
Q4: 这条信息出现过 3 次以上？                  YES → L2 行为层（晋升）
Q5: 这条信息 6 个月后还有参考价值？             YES → L2 行为层（新建）
以上全 NO → L1 情境层
```

附加判定：
```
有明确成功/失败结果评估？  YES → ai-space/L3-experience/
直接矛盾已有条目？         YES → 追加演化记录到原条目末尾 + 新建冲突卡片（不写入原层，状态标注 [认知冲突·待验证]）
```

## 晋升路径（不可修改）

```
L1 情境层  ──（同一模式出现 ≥3 次）──▶  L2 行为层
L2 行为层  ──（附带明确的"为什么"）──▶  L3 认知层
L2 工作流  ──（确认次数达到 3 次）  ──▶  Skill 缺口候选（→ 告知用户确认）
L3 认知层  ──（跨多个不同项目成立）──▶  L4 候选（→ 告知用户确认）
```
