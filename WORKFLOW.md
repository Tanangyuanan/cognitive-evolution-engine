# WORKFLOW.md — 流程：如何执行7个阶段

> **更新频率**：执行逻辑变更时更新（中等稳定性）
> **回答的问题**：触发后，按什么步骤完成整个记忆提炼？

---

## 执行总览

触发 → Phase 1（读取准备）→ Phase 2（信号提取）→ Phase 3（三步判定）→ Phase 4（写入）→ Phase 5（Skill检测）→ Phase 6（进化更新）→ Phase 7（日报生成）

**全程静默。唯二人工确认点：Phase 5的Skill缺口 + Phase 4的L4首次写入。**

---

## Phase 1：启动读取

**目标**：理解当前状态，建立写入基准。

按顺序读取：
1. `system/reading-weights.md` — 当前各层权重，确定本次重点扫描层级
2. `asset-space/L4-core/values.md` — 核对宪法条款，避免矛盾写入
3. 按权重读取对应层文件（权重 ≥1.1 的层级优先完整读取）

**静默处理**：读取失败（文件不存在）→ 跳过，继续执行，在日报末尾注明"[文件缺失：路径]"。

---

## Phase 2：信号提取

**目标**：从对话中识别值得写入的信号。

### 高价值信号识别表

| 信号表述 | 类型 | 目标层级 |
|----------|------|----------|
| "我一般/通常/总是这样做" | 行为模式 | L2-behavior |
| "按上次那样做" | 工作流重复 | L2-behavior（计数+1） |
| "哦原来如此" / "我没想到" | 高预测误差（Friston） | L3-cognitive |
| "我之前想错了" / "原来逻辑是" | 双环学习（Argyris） | L3-cognitive |
| "有没有论文/框架参考" | 知识锚定需求 | L3-cognitive/domain-knowledge |
| "这个项目我在做X" | 当前情境 | L1-context |
| "以后我想/打算" | 意图（时效性强） | L1-context |
| 某个操作成功完成 | AI经验 | ai-space/L3-experience/success-cases |
| 某个操作遇到阻塞 | AI坑点 | ai-space/L3-experience/failure-cases |

**低价值信号（跳过，不写入）**：
- 闲聊、问候、简单确认
- 已有卡片的重复表述（语义重叠 >80%）
- 对话中已经被用户纠正/否定的内容

---

## Phase 3：三步新认知判定协议

**每条提炼出的信号，按顺序执行以下三步。**

### STEP 1 — 矛盾检测（防暗示性）

**问**：这条新信息与记忆系统中已有的任何条目是否存在直接矛盾？

- **无矛盾** → 进入 STEP 2
- **有矛盾** → **不覆盖原条目**，在原条目末尾追加演化记录：
  ```
  [认知演化 YYYY-MM-DD] 出现矛盾信号：[新证据的简短描述]
  演化前：[原有认知摘要]
  演化后：[新认知摘要]
  待后续验证。
  ```
  然后**同时将新信息作为新卡片写入**，状态标注 `[认知冲突·待验证]`。

### STEP 2 — 语义距离测试（Piaget图式理论）

**问**：这条信息与已有卡片的语义重叠度是多少？

- **>80% 重叠（同化）** → 不新建卡片，在已有卡片元数据追加：
  ```
  [confirmed: YYYY-MM-DD] 原文锚点：> "新引用"
  ```
  确认次数 +1。

- **40-80% 重叠（扩展）** → 在已有卡片的"具体例证"部分追加新例子，不新建。

- **<40% 重叠（轻度调适）** → 新建卡片，状态标注 `[新认知·待验证]`。

- **直接矛盾（已在STEP 1处理）** → 按STEP 1的演化日志处理。

### STEP 3 — 抽象层级测试（Argyris双环学习）

**问**：这条信息描述的是"做了什么"还是"为什么这样想"？

- **描述行为/工作流（单环学习）** → 写入 `L2-behavior/`
- **描述认知框架/思维模型（双环学习）** → 写入 `L3-cognitive/`
- **首次出现、时效性强** → 写入 `L1-context/`
- **跨3个不同项目已验证** → `L4-core/`候选（须人工确认）

**歧义处理**：无法判断时，优先写入 L2，在元数据注明"[待升级到L3·需确认]"。

## Phase 4：写入记忆系统

**格式**：所有写入严格遵循 CARD-FORMAT.md 的 Zettelkasten 格式。

**写入规则**：
- 新建卡片：追加到对应文件末尾，用 `---` 分隔
- 更新已有卡片：仅修改"确认次数"和追加"[confirmed: ...]"，不改其他内容
- L4首次写入：**此处须人工确认**，写入前输出："[L4候选] 以下内容准备写入核心层，请确认：[卡片内容摘要]"

**来源标注**（必填，见 MEMORY-MAP.md）：
- `[用户原话]` / `[Claude提炼]` / `[共同确认]`，三选一，不可省略。

---

## Phase 5：Skill 缺口检测

**检测规则**：统计本次对话中出现的工作流模式：
- 同一工作流在 `workflows.md` 中确认次数达到 **3次** → 触发Skill缺口候选
- 触发后**必须询问用户**（这是唯一人工确认点之一）：
  ```
  [Skill缺口候选] 检测到工作流"[名称]"已出现3次，建议抽象为新Skill。
  是否需要现在创建？（可回复"是"/"否"/"稍后"）
  ```
- 用户回复"否"或"稍后" → 在 `workflows.md` 对应卡片状态标注 `[Skill候选·用户暂缓]`，不再提醒，直到再次出现2次。

---

## Phase 6：系统自进化

**目标**：根据本次对话的层级活跃度，更新读取权重。

**权重更新算法**（详见 EVOLUTION.md）：
1. 统计本次对话各层新建/更新的卡片数量
2. 最活跃层权重 +0.1，未触及层权重 -0.05（下限 0.5，上限 2.0）
3. 将新权重写入 `system/reading-weights.md`
4. 同时更新 `session_count` +1 和 `dominant_layer`

---

## Phase 7：生成会话日报

**目标**：将本次对话完整记录为可独立阅读的日报。

**格式**：严格遵循 CARD-FORMAT.md 的「会话日报格式」。

**写入位置**：`ai-space/L1-memory/daily/YYYY-MM-DD.md`

**归档规则**：
- 7天前的日报自动标注 `[已归档]`（仅标注，不删除）
- 30天前的 L1-context 条目标注 `[stale·待清理]`

**完成标志**：日报文件写入完成即为执行结束��无需任何总结输出给用户（静默协议）。
